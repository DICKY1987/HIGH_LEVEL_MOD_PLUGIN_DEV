# Coding assistants & code-automation (CLI-first)

* **Aider** — terminal pair-programmer that edits files directly in your repo from chat.
* **Continue** — open-source coding assistant with CLI/headless modes for local workflows.
* **SWE-agent** — research-grade CLI agent focused on software-engineering tasks and evals.
* **AutoCodeRover** — CLI/code agent that proposes and applies minimal code changes.
* **GPT-Engineer** — generates or upgrades whole codebases from specs via CLI prompts.
* **MetaGPT** — multi-agent “virtual company” that turns requirements into repos.
* **Claude Code (CLI)** — Anthropic’s repo-aware assistant that can read/edit/run code locally.
* **GitHub Copilot (CLI)** — terminal helper for code suggestions and shell/Git workflows.
* **Open Interpreter** — runs code and local tools safely from natural-language commands.
* **OpenHands (formerly OpenDevin)** — dev agent with full CLI and headless execution.
* **Jules Tools** — command-line companion for Google’s Jules agent to start/track tasks.

# Multi-agent frameworks & runtimes

* **AutoGen** — Python framework for orchestrating cooperating LLM agents.
* **CrewAI** — build “crews” of role-specialized agents with a simple Python API/CLI.
* **LangGraph** — graph-based runtime for controllable, stateful LLM workflows.
* **SuperAGI** — OSS agent platform with tools, UI, and headless automation.
* **Dify** — open platform for LLM apps with workflows, webhooks, and CLI deploys.
* **CAMEL** — research framework for role-playing/multi-agent simulations.
* **ChatDev** — simulates a software company of agents to plan/build projects.
* **BabyAGI** — minimalist autonomous agent loop for headless task execution.
* **GPTScript** — write agentic workflows as scripts and run them via CLI.
* **AgentScope** — multi-agent runtime with Docker/K8s deploy options.
* **Lagent** — lightweight, Apache-licensed agent framework/library.
* **OpenAI Swarm** — minimal Python library for multi-agent coordination.
* **Langroid** — research-backed multi-agent toolkit for tool use and dialogue.

# Memory / stateful agents

* **MemGPT / Letta** — long-term memory agents with a `memgpt`/`letta` CLI for stateful tasks.

# RAG / data & knowledge tooling

* **LlamaIndex** — framework to build RAG pipelines over local or remote data sources.
* **Ragna** — open-source RAG orchestrator with a simple `ragna` CLI and TOML configs.
* **Cognee** — “AI memory” layer to ingest, index, and query knowledge in pipelines.

# Spec / governance / design-token tooling

* **OpenSpec** — spec-driven dev toolkit to init/validate/version project specs via CLI.
* **Specify** — design-token engine with CLI to sync tokens from design tools to code.

# Model CLIs & local runners

* **LLM (Simon Willison)** — universal CLI to prompt many models (APIs or local).
* **Ollama** — local LLM runtime to pull and run models entirely on your machine.

# Foundation models & SDKs

* **Gemini** — Google’s family of multi-modal/code-capable models and SDKs.
* **DeepSeek** — efficient open models (e.g., DeepSeek-Coder) optimized for coding/reasoning.
* **OpenAI Codex (legacy)** — early code-generation model that powered initial code assistants.

# System / desktop agents

* **Bytebot** — self-hosted desktop agent that operates a full Linux session via container.

# Other / early platforms

* **Lemon AI** — Docker-based OSS agent platform aimed at headless automation.

# MCP servers & related

* **Ruff MCP** — MCP server exposing fast Python lint/format/fix operations.
* **Analyzer (Ruff+Vulture)** — MCP that combines style/error linting with dead-code checks.
* **Python LFT (Lint/Format/Test)** — single MCP to run lint, format, and tests on repos.
* **Pylint MCP** — MCP wrapper to run Pylint with structured results.
* **Pytest runners** — MCP tools to execute tests and return machine-readable reports.
* **PyPI package intel** — MCP to query package versions, deps, and metadata.
* **GitMCP** — turns GitHub repos into an MCP knowledge hub with precise code context.
* **Sourcegraph codesearch MCP** — MCP for cross-repo code search/snippets.
* **Git operations MCP** — MCP utilities for commits, branches, diffs, and grep.
* **PowerShell Exec MCP** — execute PowerShell commands/scripts via MCP safely.
* **PowerShell MCP (general)** — broader PowerShell MCP for discovery and script runs.
* **PSScriptAnalyzer (via MCP tool)** — standard PS linter exposed through a PS MCP tool.
* **Microsoft AI Shell** — PowerShell-first shell for AI workflows and tool hosting.
* **PowerShell Universal** — host dashboards/APIs/automation; useful for MCP + PS ops.

# Core runtimes & CLIs

* **Python 3.12** — modern CPython with performance and typing upgrades for apps/agents.
* **Node.js LTS** — long-term-support JavaScript/TypeScript runtime for CLIs and tools.
* **PowerShell 7** — cross-platform automation shell and scripting language.

# Linters / formatters / type checkers

* **Ruff** — ultra-fast Python linter/formatter written in Rust.
* **Black** — opinionated Python code formatter for consistent style.
* **isort** — automatically sorts and groups Python imports.
* **Pylint** — comprehensive static analysis with style and bug checks.
* **mypy** — optional static type checker using Python type hints.
* **pyright** — fast, scalable type checker and language server for Python.
* **Semgrep** — multi-language static analysis with custom rule support.
* **Gitleaks** — secrets scanner for repos, directories, and pipelines.
* **yamllint** — YAML linter to catch syntax and style issues.
* **markdownlint-cli** — CLI to enforce Markdown style rules.
* **mdformat** — CommonMark-compliant Markdown formatter.
* **codespell** — spell checker for source and docs.
